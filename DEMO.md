# Sentinel AI Demo

This is the two-minute flagship demo path.

## Setup

Start Sentinel:

```bash
./mvnw -f backend/pom.xml spring-boot:run
```

Or run the Postgres-backed stack:

```bash
docker compose up --build
```

Open:

```text
http://localhost:8090
```

For the polished first impression, start at:

```text
http://localhost:8090/landing.html
```

Demo login:

```text
release@sentinel.ai / sentinel-release
```

Second tenant login for isolation demos:

```text
acme-release@sentinel.ai / sentinel-acme
```

## Talk Track

1. **Sentinel wakes up**
   - Open the landing page, then launch the command center.
   - Show the AI Executive Briefing.
   - Point to the Demo Mode checklist in the rail.
   - Say: “Sentinel is not a dashboard. It is an AI Chief Engineer that starts with judgment.”

2. **Sentinel reviews the engineering organization**
   - Point to the tenant workspace card, Chief Engineer Brain provider, deployments, high-risk releases, Engineering DNA, and projected savings.
   - Open Production Integrations and connect GitHub, Jira, and CI.
   - Ask: “What deployments are unsafe right now?”

3. **Sentinel remembers history**
   - Select `payment-api`.
   - Show AI Memory.
   - Say: “This is the moat: Sentinel remembers service patterns, incidents, dependencies, and prior decisions.”

4. **Sentinel reviews a pull request**
   - Use AI Engineer mode.
   - Simulate PR `#418`.
   - Ask: “Should I merge this pull request?”
   - Record `WAITING` or `BLOCKED`.

5. **Sentinel understands architecture**
   - Open Architecture Brain.
   - Ask: “Where is our architecture most fragile?”
   - Show blast radius, shared database, and circular dependency findings.

6. **Sentinel controls release risk**
   - Simulate a GitHub deployment signal.
   - Attach CI evidence and Jira work context.
   - Show the new deployment, memory, and risk explanation.
   - Block the release.
   - Show the audit trail.

## One-Command API Demo

With the app running:

```bash
scripts/demo.sh
```

Optional:

```bash
BASE_URL=http://localhost:8090 scripts/demo.sh
```

## What The Demo Proves

- JWT auth works.
- Tenant-scoped integration installation state works.
- Integration health and sync history are visible.
- App-level security headers and rate limiting are active.
- Executive briefing is generated by backend intelligence APIs.
- Chief Engineer reasoning is routed through a provider abstraction.
- Durable memory links deployments to engineering events.
- AI Engineer reviews PRs and records decisions.
- Architecture Brain detects service graph risk.
- GitHub webhook simulation creates a deployment review.
- CI and Jira ingestion attach build/test/work evidence to that review.
- Approval/block decisions are audited.
